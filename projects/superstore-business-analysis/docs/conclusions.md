# Superstore 经营分析结论

## 1. 整体经营规模扩大，但利润质量需要继续看结构

发现：数据共 9994 行，订单日期覆盖 2014-01-03 到 2017-12-30。总销售额为 2,297,200.86，总利润为 286,397.02，整体利润率为 12.47%。2017 年销售额最高，为 733,215.26；2017 年利润也最高，为 93,439.27。

可能原因：订单规模扩大带来了更高销售额和利润，但不同月份、区域、品类和折扣策略对利润的影响不同。只看年度总销售额，不能判断增长质量。

建议：经营总览中必须同时看销售额、利润和利润率。后续月报应保留月度环比，避免只用年度汇总掩盖短期利润波动。

## 2. 利润问题集中在部分区域和子品类

发现：Central 区域利润率最低，为 7.92%。Furniture 大类利润率最低，为 2.49%。子品类中 Tables 总利润为 -17,725.48，Bookcases 为 -3,472.56，Supplies 为 -1,189.10。

可能原因：这些子品类可能存在折扣过高、定价偏低、履约成本偏高或产品结构不合理的问题。当前数据没有成本和物流费用，因此只能定位利润异常，不能直接解释全部原因。

建议：优先对 Tables、Bookcases、Supplies 做专项复盘，检查对应区域、折扣水平和订单结构。不要只按销售额排名决定资源投入。

## 3. 高折扣是最明确的利润风险

发现：无折扣订单利润率为 29.51%。折扣超过 30% 的订单销售额为 259,543.49，但利润为 -125,006.78，利润率为 -48.16%，亏损率为 97.77%。高折扣下亏损最严重的子品类是 Binders，利润为 -38,510.50，利润率为 -106.56%。

可能原因：高折扣订单虽然带来销售额，但折扣侵蚀了利润。一些子品类在高折扣下几乎必然亏损，说明促销策略没有利润底线。

建议：对 `discount > 0.3` 的订单设置审批或预警，尤其关注 Binders、Machines、Tables、Bookcases。促销复盘不能只看销售额提升，必须同时看利润率和亏损率。

## 4. 高消费客户不一定是高价值客户

发现：利润贡献最高的客户是 Tamara Chand，销售额 19,052.22，利润 8,981.32，利润率 47.14%。销售额最高的客户是 Sean Miller，销售额 25,043.05，但利润为 -1,980.74，利润率 -7.91%。高销售但亏损客户中，Cindy Stewart 利润为 -6,626.39，Grant Thornton 利润为 -4,108.66，Sean Braxton 利润为 -2,082.75。

可能原因：部分客户消费金额高，但订单折扣、商品结构或履约成本可能导致利润为负。仅用销售额识别大客户，会把低质量客户误判为高价值客户。

建议：客户运营要同时看销售额、利润、折扣和购买频次。高销售亏损客户需要单独复盘，不应继续无条件让利。

## 5. RFM 分层适合做客户运营的第一版框架

发现：SQL 已使用 `recency_days`、`frequency`、`monetary` 分别打 1-5 分，并将客户分为高价值客户、重点发展客户、新近低频客户、流失风险客户、高消费低利润客户、低价值沉默客户和普通客户。

可能原因：RFM 能把客户最近购买、购买频次和消费金额放到同一个框架中，比单独看消费总额更稳定。但 Superstore 数据没有会员注册、访问、营销触达和复购周期信息，因此 RFM 仍是初步分层。

建议：将 RFM 结果作为客户运营清单的第一版。对高价值客户做维护，对流失风险客户做召回，对高消费低利润客户收紧折扣策略。

## 6. 数据限制

当前数据没有成本明细、物流费用、库存、营销活动、广告投放、客户访问和注册数据。因此不能直接分析真实毛利、库存周转、广告 ROI、获客来源或长期留存。

这个项目的结论应定位为经营诊断和问题定位，不应写成完整因果解释。
