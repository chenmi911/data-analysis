# 分析过程说明

## 1. 为什么先做字段统一

这个数据集有 5 个 CSV 文件，但不同年份字段命名不一致。如果直接 `concat`，会得到多组语义相同但名称不同的字段，例如：

| 标准字段 | 2015/2016 | 2017 | 2018/2019 |
|---|---|---|---|
| country | Country | Country | Country or region |
| rank | Happiness Rank | Happiness.Rank | Overall rank |
| score | Happiness Score | Happiness.Score | Score |
| gdp | Economy (GDP per Capita) | Economy..GDP.per.Capita. | GDP per capita |
| social_support | Family | Family | Social support |

所以脚本先用 `RENAME_MAPS` 显式建立字段映射，再合并成长表。

## 2. 为什么只保留 11 个核心字段

本项目的目标是训练基础数据分析流程，而不是覆盖所有统计字段。部分字段并不是每年都有，例如置信区间、标准误、Dystopia Residual。为了保证跨年分析口径一致，本项目保留：

```text
year, country, region, rank, score, gdp, social_support,
life_expectancy, freedom, generosity, corruption
```

这个选择牺牲了一部分信息，但换来了更清楚的分析口径。

## 3. 为什么用 2015/2016 补地区

2015 和 2016 有 `Region` 字段，2017-2019 没有完整地区字段。脚本先从有地区字段的年份建立国家到地区的映射，再给后续年份补齐：

```python
clean["region"] = clean["region"].fillna(clean["country"].map(region_map))
```

补齐后仍有 8 条地区缺失，主要原因是国家名称写法不一致。第一版分析不做静默手工修正，避免引入未说明的数据加工。

## 4. 为什么比较变化时只取共同国家

2015 和 2019 的国家集合并不完全相同。直接比较两个年份全量数据会把新增、缺失国家混在一起，导致变化分析不严谨。

正确做法是：

1. 取 2015 和 2019 共同出现的国家。
2. 转成宽表。
3. 计算 `2019 - 2015`。

这样得到的变化量才是同一国家在两个年份之间的变化。

## 5. 为什么相关性不是因果

相关系数只能说明两个变量共同变化的方向和强度，不能证明一个变量导致另一个变量变化。本项目中 GDP、健康预期寿命、社会支持与幸福分数相关性较高，但不能写成“GDP 提高导致幸福分数提高”。

更稳妥的表达是：

- GDP 与幸福分数高度正相关。
- 健康预期寿命较高的国家通常幸福分数也较高。
- 这些关系可能反映经济、公共健康和社会支持体系与居民幸福感之间的联系，但需要更多数据验证。

## 6. 企业分析中的对应能力

这个项目练的是企业数据分析的基础动作：

| 本项目动作 | 企业常见场景 |
|---|---|
| 字段统一 | 多月报表、多系统数据口径统一 |
| 缺失值检查 | 指标异常排查、数据质量监控 |
| Top/Bottom | 门店、商品、地区、客户表现排名 |
| 趋势分析 | GMV、利润、活跃用户、库存变化 |
| 分组对比 | 地区、渠道、品类、用户层级对比 |
| 变化量分析 | 同比、环比、活动前后对比 |
| 相关性分析 | 初步判断指标关系 |
| 可视化 | 周报、月报、经营看板、管理层汇报 |
